## Process Creation

The OS must load the program code and static data into memory from disk. Early operating systems would eagerly load the entire program and data into memory. Modern operating systems perform lazy loading, loading pieces of code or data only when needed during program execution.

The OS must allocate memory for the program's stack. The stack is used for local variables, function parameters, and return addresses. The OS also initializes the stack with arguments to the main() function (argc and argv).

The OS must allocate memory for the program's heap. The heap is used for dynamically allocated data. Programs request heap memory using malloc() and free it using free(). The heap starts small and grows as needed.

The OS performs I/O initialization by creating file descriptors. In UNIX systems, three file descriptors are opened by default: standard input, standard output, and standard error.

Finally, the OS transfers CPU control to the entry point, typically the main() function, and the program begins execution.
